# ADS-B Wi-Fi Manager

**JLBMaritime - Raspberry Pi 4B**

A complete ADS-B receiver and forwarder system with Wi-Fi hotspot management, web interface, and CLI tool. Designed for unattended, long-term operation on a Raspberry Pi 4B.

---

## ✨ Features

### **ADS-B Server**
- Receives ADS-B data from FlightAware SDR via dump1090-fa
- Filters aircraft by ICAO IDs (configurable, or all aircraft)
- Forwards SBS1 format data to multiple TCP destinations
- Configuration file: `adsb_server_config.conf`
- Resource leak fixes (sockets, threads, memory)
- Resource monitoring every 5 minutes
- Logging with 72-hour auto-purge

### **Wi-Fi Manager**
- Web interface accessible at `http://ADS-B.local`
- Hotspot on `wlan1` (SSID: `JLBMaritime-ADSB`, Password: `Admin123`)
- Internet connection via `wlan0`
- Scan, connect, and forget Wi-Fi networks
- Network diagnostics with ping test
- Mobile-friendly responsive design
- Login: `JLBMaritime` / `Admin`

### **CLI Tool**
- Run `adsb-cli` from anywhere
- Manage services, view logs, configure ADS-B
- Wi-Fi management via terminal
- System status and diagnostics

### **Stability & Auto-Recovery** 🛡️
- **Hardware Watchdog** — Pi auto-reboots if frozen (no power cycle needed!)
- **Service auto-restart** — Services restart automatically on crash
- **Memory limits** — Services killed and restarted if they exceed 300 MB
- **Health monitor** — Runs every 10 minutes, auto-restarts dead services
- **Resource leak fixes** — Socket timeouts, thread limits, proper cleanup
- **Log rotation** — Prevents SD card from filling
- **SD card optimizations** — `noatime`, reduced swappiness
- **No random reboots** — Unattended upgrades disabled

## 🚀 Installation (Fresh Raspberry Pi)

### Prerequisites
- Raspberry Pi 4B (2 GB or more)
- Raspberry Pi OS Bookworm (64-bit, **no desktop**)
- Two Wi-Fi adapters (`wlan0` for internet, `wlan1` for hotspot)
- FlightAware SDR USB stick + ADS-B antenna
- Official Raspberry Pi 3A power supply
- Username: `JLBMaritime`, Hostname: `ADS-B`

### One-Command Install

```bash
git clone https://github.com/YOUR_USERNAME/adsb-wifi-manager.git
cd adsb-wifi-manager
sudo ./install.sh
sudo reboot
```

That's it! After reboot, the system will be running with:
- Hotspot active on `wlan1`
- Web interface at `http://ADS-B.local`
- ADS-B server forwarding data
- Hardware watchdog protecting against freezes

## 📡 First Use

### **1. Connect to the Hotspot**
Connect any device (phone/laptop) to:
- **SSID:** `JLBMaritime-ADSB`
- **Password:** `Admin123`

### **2. Open the Web Interface**
- URL: `http://ADS-B.local` (or `http://192.168.4.1`)
- **Username:** `JLBMaritime`
- **Password:** `Admin`

### **3. Configure Internet Wi-Fi**
- Go to **Wi-Fi Manager** tab
- Scan for your home network
- Click **Connect** and enter the password
- The Pi will use this Wi-Fi for internet

### **4. Configure ADS-B Endpoints**
- Go to **ADS-B Configuration** tab
- Set destination IP and port
- Configure ICAO filter (specific IDs or "ALL")
- Save and restart the service

## 🩺 Troubleshooting

### Cannot reach `ADS-B.local`
```bash
# Check hotspot
sudo systemctl status hostapd dnsmasq

# Try direct IP
http://192.168.4.1
```

### ADS-B server not receiving data
```bash
# Check dump1090-fa
sudo systemctl status dump1090-fa
ls /dev/ttyUSB* /dev/sdr*

# View logs
sudo journalctl -u adsb-server -f
```

### Services keep restarting
```bash
# Check memory usage
sudo systemctl status adsb-server | grep Memory

# If hitting 300MB limit, edit:
sudo systemctl edit adsb-server
# Add: [Service]
#      MemoryMax=500M
```

### Hardware watchdog not working
```bash
# Check device exists
ls -l /dev/watchdog

# Check config
grep watchdog /boot/firmware/config.txt
# Should show: dtparam=watchdog=on

# Reboot if missing
sudo reboot
```

### System still freezing (rare with V2)
```bash
# Check throttling (power/thermal)
vcgencmd get_throttled
# 0x0 = OK; anything else = power supply or thermal issue

# Check temperature
vcgencmd measure_temp
# Should be < 80°C

# Check kernel errors from before crash
sudo journalctl -k -b -1 | tail -100
```

## 📊 Stability Architecture

```
┌─────────────────────────────────────┐
│  Layer 1: Hardware Watchdog (15s)   │ ← Reboots Pi if frozen
├─────────────────────────────────────┤
│  Layer 2: Software Watchdog          │ ← Monitors load/memory
├─────────────────────────────────────┤
│  Layer 3: systemd Service Limits     │ ← Memory, FDs, tasks
├─────────────────────────────────────┤
│  Layer 4: systemd Auto-Restart       │ ← On any crash
├─────────────────────────────────────┤
│  Layer 5: Health Monitor (10 min)    │ ← Auto-recover services
├─────────────────────────────────────┤
│  Layer 6: Application-Level Fixes    │ ← Timeouts, leak prevention
└─────────────────────────────────────┘
```

**Result:** Even if any single layer fails, others ensure recovery. Maximum downtime per freeze: ~60 seconds (vs. waiting for manual power cycle).
